# Contributing Guide

How to contribute to RustChain and earn RTC.

## Ways to Contribute

| Type | Reward | How |
|------|--------|-----|
| Bug reports | 1-5 RTC | File detailed issues with reproduction steps |
| Code fixes | 5-50 RTC | Submit PRs to Scottcjn/Rustchain |
| Documentation | 10-25 RTC | Improve docs, write guides |
| Content creation | 25-100 RTC | Articles, videos, tutorials |
| Community | 1-5 RTC | Reviews, engagement, onboarding |

## Development Setup

```bash
git clone https://github.com/Scottcjn/Rustchain
cd Rustchain
pip install -r requirements.txt
```

See [BUILD.md](../BUILD.md) for full build instructions.

## Pull Request Process

1. Fork the repository
2. Create a feature branch: `git checkout -b feat/my-feature`
3. Make your changes
4. Run tests and build locally
5. Submit PR with clear description
6. Include your RTC wallet in PR description

## Bounty Program

Active bounties: https://github.com/Scottcjn/rustchain-bounties/issues

To claim a bounty:
1. Find an open issue labeled `bounty`
2. Complete the task as described
3. Comment on the issue with proof and your RTC wallet
4. Maintainer reviews and approves
5. RTC appears in your wallet after review

## Code Style

- Python: 4 spaces (PEP 8)
- Markdown: semantic headers, code blocks
- Commit messages: conventional format (`feat:`, `fix:`, `docs:`)

## Community Standards

- Be respectful and constructive
- Provide evidence for bounty claims
- Write clear, reproducible bug reports
- Test changes before submitting

## Recognition

Active contributors earn:
- RTC tokens per contribution
- Leaderboard placement
- Miner tier status
- Wall of Miners listing at rustchain.org
